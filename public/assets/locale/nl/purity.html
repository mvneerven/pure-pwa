• Alle Standaard PWA Voordelen
    • Kosteneffectiviteit
    • Geen App Store Bemiddelaars
    • Offline Functionaliteit
    • SEO Voordelen
    • Lagere Bounce Rates
    • Snellere Installatie
    • Cross-Platform Compatibiliteit
    • Bijna-Native Mogelijkheden
    • Eenvoud van Ontwikkeling - gewoon nog een Web App
    • Toegankelijkheid
    • Inspannings- & Kostenbesparing, door Enkele Codebase
    • Automatische Updates
    • Veel lagere Dataverbruik dan een Native App
  • Hybride MPA/SPA Voordelen
    • Extra SEO-vriendelijkheid, doordat er meerdere pagina's zijn die afzonderlijk kunnen worden geïndexeerd
    • Veel minder client-side JavaScript-rendering
    • Pagina-specifieke code wordt automatisch geladen wanneer nodig, zonder de complexiteit die wordt geïntroduceerd in SPA-apps (zoals tree-shaking, bundle-splitting & lazy-loading)
    • Ideaal voor Server-side Rendering en JAMStack frameworks
    • Uiterst schaalbaar MPA-model
    • Gedeelde Webcomponenten voor app-brede of multi-use functionaliteit/styling
    • SPA-gedrag waar nodig, met behulp van native Navigator.navigate event-onderschepping
    • MPA paginaloads geoptimaliseerd met behulp van achtergrondcaching
    • Uiterst Ontdekbaar
  • Native Ervaring
    • Functionele micro-animaties
    • Scroll-snapping
    • View transitions
    • Titelbalk onderklasse gebruikt de Window Controls Overlay API
    • PWA Snelkoppelingen
    • Mobile-first, responsive design
    • Gebruik van CSS Grid + Flex
    • Geen concessies op Mobiel - Touch-vriendelijk, Menubalk onder
    • Media queries alleen wanneer niets anders werkt.
    • Hoogst mogelijke Prestaties (Lighthouse, Core Web Vitals)
  • Geen Afhankelijkheden
    • Geen framework, en geen enkele bibliotheek. Gewoon het Pure, Native Web
    • Profiteren van de rijke set Native Mogelijkheden die het Moderne Web te bieden heeft, zoals CSS animaties/transities
    • Eenvoudig, gemakkelijk te begrijpen CustomElement base class
    • Native State Management met Proxy en Custom DOM events
  • Geen Build, Geen Bundling
    • Alleen gebruik van Runtime ES Imports.
    • Geen framework-'bloat'
    • Geen NPM afhankelijkheidshel
    • Veel kleinere, native stukjes JavaScript worden alleen geladen wanneer dat nodig is
  • Ontwikkelaarsvriendelijk
    • Hoogst ontdekbare, gemakkelijk leesbare code, dus gemakkelijke onboarding
    • Gebruik van ES-modules die over pagina's kunnen worden gedeeld of geïsoleerd kunnen worden tot MPA paginascope
    • Elke MPA-route heeft zijn eigen HTML, CSS en JavaScript (in een aparte map)
    • Gebruik van Semantische HTML Structuren die betekenis/intentie tonen aan ontwikkelaars
    • Zero Setup. Geen NPM, geen Transpiling, geen Bundling. Start gewoon een Webserver op /public/
  • SVG Sprite Sheets
    • Een enkel SVG Sprite Sheet bestand
    • Meerdere herbruikbare Symbolen gedefinieerd
    • Geen behoefte aan Glyph Web Fonts die moeten worden gebouwd
    • Veel efficiënter dan individuele Icon Afbeeldingen
  • Toekomstbestendig
    • Geen afhankelijkheden van Derde-Partij Frameworks betekent geen externe opgelegde Techn Debt
    • Het Web zal altijd Backwards Compatibel zijn
    • Progressive Enhancement kan hiaten opvullen maar kan in de loop van de tijd worden verwijderd
